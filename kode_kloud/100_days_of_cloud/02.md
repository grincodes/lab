# Quest
-Create a security group with name devops-sg description QuestSecurityGroup
* Add the inbound rule of type HTTP with port range of 80.
Enter the source CIDR range of 0.0.0.0/0
* Add another inbound rule of type SSH with port 22 and CIDR range of 0.0.0.0/0

# Process and Solution
- Use this Aws cli command to create security group
  ```aws ec2 create-security-group --group-name "devops-sg" --description "QuestSecurityGroup"```
- Set Ingres Inbound on security group with this command
    ``` aws ec2 authorize-security-group-ingress \ 
        --group-id <sg-xxxxx : gotten from the above cmd> \
        --ip-permissions \
          IpProtocol=tcp,FromPort=22,ToPort=22,IpRanges='[{CidrIp=0.0.0.0/0,Description="SSH access"}]' \
          IpProtocol=tcp,FromPort=80,ToPort=80,IpRanges='[{CidrIp=0.0.0.0/0,Description="HTTP access"}'
    ```
